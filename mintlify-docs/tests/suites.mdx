title: "UsbUiTest suite manifests: run groups of tests together"

A suite is a YAML manifest that groups individual test files into a logical collection you can run together. Rather than running tests one at a time, you define a suite for each feature and run every scenario in it with a single command. Suite manifests live under `.usb-ui-test/suites/`.

## Suite fields

<ParamField path="name" type="string" required>
  A stable identifier for the suite. Use `snake_case`. This name is what you pass to `usb-ui-test suite` when you want to run it.
</ParamField>

<ParamField path="description" type="string">
  A short, human-readable summary of what the suite covers. One or two sentences is enough.
</ParamField>

<ParamField path="tests" type="list of strings" required>
  An ordered list of test file paths. Each path is relative to `.usb-ui-test/tests/`. The agent runs the tests in the order listed.
</ParamField>

## Example suite

```yaml
name: auth_smoke
description: Covers the authentication smoke scenarios.
tests:
  - auth/login.yaml
  - auth/logout.yaml
```

## Organizing suites by feature

The recommended convention is one suite per feature folder, mirroring the structure of `.usb-ui-test/tests/<feature>/`. A suite named `auth_smoke` covering tests in `.usb-ui-test/tests/auth/` is a clear, predictable mapping that makes it easy to find which suite runs a given test.

<Tip>
  Name your suite files after their feature folder. If your tests live in `.usb-ui-test/tests/checkout/`, name the suite file `.usb-ui-test/suites/checkout.yaml` and give it the `name: checkout` identifier. This one-to-one convention keeps suites discoverable as the test library grows.
</Tip>

## Running a suite

Pass the suite manifest path to `usb-ui-test suite`. Specify a platform and AI model:

```bash
usb-ui-test suite auth_smoke.yaml --platform android --model google/gemini-3-flash-preview
```

You can also target iOS:

```bash
usb-ui-test suite auth_smoke.yaml --platform ios --model google/gemini-3-flash-preview
```

## Validating a suite before running

Run `usb-ui-test check` with the `--suite` flag to validate the suite manifest and all referenced test files — confirming that paths resolve, placeholders are declared, and the workspace is configured correctly — before spending time on a full test run:

```bash
usb-ui-test check --suite auth_smoke.yaml
```

Fix any errors reported by `usb-ui-test check` before proceeding. The command output is the source of truth for binding correctness and path resolution.
